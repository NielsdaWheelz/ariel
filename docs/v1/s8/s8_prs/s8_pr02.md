# PR-02: Shared-Content Capture Hardening + Source/Policy Safety

## Goal

Complete Slice 8 by adding shared text-content capture semantics that preserve note/source separation and keep capture-origin content inside Ariel's existing taint, approval, provenance, and memory rules.

## Builds On

PR-01.

## Acceptance

- `POST /v1/captures` accepts shared text-content captures in addition to existing text/url capture modes while preserving the raw submitted payload for audit, replay, and future extension.
- Normalized turn input preserves explicit separation between user-authored note and shared source material so bare shared content is observe-first context, not an implicit command, approval, or direct memory instruction.
- Shared source material is treated as untrusted ingress provenance for the resulting turn, so side-effecting proposals influenced by it require approval or are denied under the same taint rules that already govern other untrusted content.
- Shared-content capture turns can use normal browsing/retrieval flows, and any externally grounded assistant claims still surface citations and artifact-backed provenance through the existing turn response and timeline contracts.
- Capture outcomes continue to distinguish ingress rejection from post-ingest turn failure: invalid or unsupported shared-content payloads fail durably before turn creation, while downstream turn failures retain capture-to-session/turn linkage with typed error surfacing.
- Shared-content capture retries remain capture-scoped and replay-safe across session rotation, including conflict handling when the same idempotency key is reused with different content.
- Shared-content capture turns can create, promote, correct, or retract memory only through the existing conversation-mediated lifecycle; raw shared payload and metadata cannot directly mutate canonical memory.
- Regression coverage blocks release on shared-content invariants: note/source separation, taint-driven side-effect blocking, durable failure classification, replay-safe idempotency, and no direct capture-to-memory side channel.

## Non-Goals

- does not add a capture-specific executor, approval bypass, or capture-only memory path outside the normal turn engine.
- does not add capture read/list endpoints or surface raw shared payload details beyond the existing capture response and linked turn/timeline inspection path.
- does not add chat-page capture UX, native mobile share-target UI, offline/background queues, batched capture submission, or binary/vision/speech capture.

## Key Decisions

**Capture-origin taint is runtime-enforced, not prompt-only**: shared source material must remain policy-tainted even if the model omits or misstates taint metadata.

**Observe-first normalization stays structurally separated**: explicit user note is the only instruction channel; shared source text and URLs remain referenced context.
