# PR-02: News + Weather Behaviors

## Goal

Complete Slice 3 by adding production-grade topic-news and weather journeys on top of PR-01 grounded retrieval, citation, and provenance foundations.

## Builds On

PR-01 merged.

## Acceptance

- topic news requests execute through `cap.search.news` as a `read` capability under allowlisted read policy (no approval), with auditable action lifecycle and explicit egress preflight enforcement.
- news responses provide relevant recent results with publication timestamps and user-visible source references (inline citations plus structured `assistant.sources[]` linked to inspectable artifacts).
- Ariel applies deterministic freshness classification for news recency and discloses stale, missing, or ambiguous publication timing instead of presenting uncertain/stale items as current.
- weather requests execute through `cap.weather.forecast` as a provider-abstracted `read` capability (SLA-backed production default backend plus non-SLA dev fallback adapter), while preserving provider-independent orchestration contracts.
- weather location resolution is deterministic and privacy-safe: explicit location first, configured default second, clarification otherwise; no implicit IP/device geolocation inference.
- configured default weather location is canonical user-managed state in Postgres with explicit set/get behavior; optional env bootstrap is first-run only when unset and never overwrites user-updated state.
- when weather location cannot be resolved, Ariel asks for clarification rather than guessing.
- weather responses include resolved location/timeframe, forecast or observation timestamps, and user-visible source references.
- upstream search/weather failures are explicit and recoverable in user-visible responses, with uncertainty/partial labeling where evidence or execution is incomplete.
- regression coverage demonstrates end-to-end Slice 3 behavior across grounded factual Q&A, news recency/attribution, weather location handling, and failure recovery without regressing PR-01 citation/provenance guarantees.

## Non-Goals

- does not add proactive scheduler or notification execution behavior.
- does not add durable-memory or session-rotation behavior changes.
- does not add provider failover guarantees beyond capability-mediated, provider-abstracted contracts.
- does not add URL extraction/full-page parsing workflows.
- does not introduce broad user-profile/settings management beyond weather-default canonical state required for deterministic weather resolution.

## Key Decisions (if any beyond L2)

**Grounded synthesis scope expands beyond web-only retrieval**: retrieval answer synthesis and citation/provenance surfacing apply consistently to retrieval-only turns that use `cap.search.web`, `cap.search.news`, and/or `cap.weather.forecast`.

**Weather default location is explicit user-owned canonical state**: this PR includes minimal but explicit weather-default set/get behavior backed by Postgres canonical state; env bootstrap seeds only initial empty state.

**Recency claims are fail-closed**: Ariel can frame news as current only when publication timing evidence passes deterministic freshness checks; otherwise it must disclose uncertainty/staleness.
