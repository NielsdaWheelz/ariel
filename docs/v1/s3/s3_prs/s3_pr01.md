# PR-01: Grounded Retrieval Core (Web Search + Provenance Artifacts + Citation-Gated Answers)

## Goal

Deliver the first Slice 3 vertical path for externally grounded factual answers via capability-mediated web retrieval, durable provenance artifacts, and citation-gated response synthesis.

## Builds On

Slice 2 PR-08 merged state.

## Acceptance

- when a factual query requires external evidence, Ariel executes `cap.search.web` as a `read` capability under policy allowlist without approval, and records an auditable action lifecycle.
- external retrieval remains provider-independent: web search executes through Ariel capability contracts rather than model-vendor native search shortcuts.
- retrieval egress is constrained to explicit allowlisted destinations; undeclared, malformed, or non-allowlisted outbound intent fails closed before outbound calls and surfaces a clear user-visible failure reason.
- Ariel returns a synthesized final answer (not raw tool-output appendix as terminal output) that includes inline citation markers plus structured `assistant.sources[]` entries linked to artifact ids.
- each cited source is persisted as an inspectable provenance artifact with stable identity and citation metadata (title, source URL/handle, retrieval time, publication time when present).
- cited provenance artifacts are retrievable through a user-facing artifact endpoint.
- Ariel does not present unsupported external factual claims as true; when evidence is missing, insufficient, or conflicting, Ariel returns explicit uncertainty and a concrete recovery step.
- retrieval timeout/rate-limit/upstream failure paths are explicit and recoverable in user responses, label partial results as partial, and remain reconstructable in action/event history.
- response and redaction boundaries remain fail-closed: surfaced citation/provenance fields are explicit allowlisted contract fields, and internal runtime-only metadata remains unsurfaced.

## Non-Goals

- does not add `cap.search.news` ranking/recency behavior.
- does not add weather capability behavior or location-resolution logic.
- does not add URL extraction/full-page parsing workflows.
- does not add cross-provider failover guarantees beyond preserving capability-mediated retrieval architecture.

## Key Decisions (if any beyond L2)

**Dual citation contract is required**: grounded responses must include both inline citation markers and synchronized `assistant.sources[]` entries referencing the same artifact ids.

**PR-01 artifact scope is retrieval provenance only**: this PR delivers inspectable source artifacts for grounded web retrieval; broader artifact-domain expansion remains in later slices.
