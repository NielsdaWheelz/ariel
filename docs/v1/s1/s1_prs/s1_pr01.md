# PR-01: Deterministic Turn Decision + Bounded Session Context

## Goal

Implement the Slice 1 conversation contract so the model chooses one conversational outcome per turn (direct answer or clarifying question) from bounded recent in-session context, with deterministic observability of that outcome.

## Builds On

Slice 0 PR-02 merged state (durable single active session + auditable turn/event chain).

## Acceptance

- when user intent is clear, Ariel returns a direct response in that turn without unnecessary clarification.
- when user intent is ambiguous or conflicting, Ariel asks one focused clarifying question instead of guessing hidden assumptions, based on model judgment from prompt+context.
- each turn uses a deterministic bounded context bundle in fixed order, with policy/system instructions applied before bounded recent active-session context.
- the configured recent-context bound applied for the turn is auditable in persisted turn/event metadata.
- follow-up turns that reference details within the configured recent-turn window remain coherent and consistent with prior in-session facts.
- when a user asks about details outside the configured context window, Ariel does not fabricate continuity and instead states uncertainty or asks for recovery details.
- completed turn events make the conversational decision outcome observable for each turn (answer vs clarification).

## Non-Goals

- does not implement turn-budget exhaustion failure semantics or `E_TURN_LIMIT_REACHED` behavior (PR-02).
- does not implement capability/tool execution, approval workflow behavior, or capability-policy authorization changes.
- does not implement cross-session memory retrieval, memory correction flows, or session-rotation behavior.
- does not implement provider failover strategy changes, multi-user tenancy, or public internet exposure behavior.
- does not introduce a brittle rule-tree intent classifier; decision policy remains model/prompt-driven within the above contracts.
