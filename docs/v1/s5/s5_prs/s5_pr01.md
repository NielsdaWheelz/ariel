# PR-01: Canonical Memory Core + Session Rotation + Lifecycle Semantics

## Goal

Deliver the shipped Slice 5 continuity vertical slice: canonical durable memory, read-only memory projection, explicit + threshold-driven session rotation, lifecycle-safe memory mutation, replay-safe ingress, and deterministic cursored timeline reads.

## Builds On

Slice 4 PR-03 merged state (bounded turn orchestration, surfaced response-contract enforcement, and policy/audit runtime already in place).

## Acceptance

- Canonical durable memory persists in Postgres with typed classes (`profile`, `preference`, `project`, `commitment`, `episodic_summary`) and revision metadata (`source_turn_id`, `source_session_id`, confidence, evidence, `last_verified_at`).
- `GET /v1/memory` exposes a strict read-only projection derived from canonical memory state; no generic memory write endpoint is added.
- Conversation-mediated memory mutation is implemented:
  - explicit `remember ...` captures validated memory,
  - `correct ...` updates memory via append-only revision,
  - `forget ...` retracts memory via append-only revision,
  - inferred statements can create `candidate` memory.
- Cross-session recall uses only active `validated` memory; `candidate` memory is excluded until explicitly promoted.
- Memory revision lifecycle is append-only and inspectable with states `candidate`, `validated`, `superseded`, `retracted`; replacing an active candidate/validated revision supersedes the prior active revision.
- Session rotation is implemented as both:
  - explicit rotate intent via `POST /v1/sessions/rotate`,
  - deterministic threshold fallback at turn boundaries (`threshold_turn_count`, `threshold_age`, `threshold_context_pressure`).
- Rotation safety/integrity is enforced:
  - exactly one active session invariant is preserved,
  - prior active session is closed when rotated,
  - typed rotation reason and trigger snapshot are recorded,
  - idempotent rotate behavior is supported through rotate idempotency keys,
  - rotation history is inspectable via `GET /v1/sessions/rotations`.
- Rotation writes bounded episodic continuity artifacts (episodic summary + open-commitment linkage) to canonical memory for auditable continuity context.
- Message ingress is replay-safe: `POST /v1/sessions/{session_id}/message` supports `Idempotency-Key`, replays identical payload outcomes, and returns `409 E_IDEMPOTENCY_KEY_REUSED` on conflicting payload reuse.
- Timeline incremental sync is deterministic and session-scoped: `GET /v1/sessions/{session_id}/events?after={event_id}` returns only events strictly after cursor, omits turns with no events after cursor, and returns `404 E_EVENT_CURSOR_NOT_FOUND` for unknown cursor.
- Turn context remains deterministic and bounded after memory integration (fixed section order, bounded recall/top-k selection, bounded recent window, and preserved turn-budget semantics).
- Regression coverage locks behavior for memory lifecycle, rotation/idempotency, context assembly, and cursored timeline semantics.

## Non-Goals

- does not add generic memory CRUD write APIs, autonomous memory rewriting, or Nexus sync-authority changes.
- does not add proactive memory-driven notification policy.
- does not introduce multi-user tenancy, background self-edit loops, or out-of-band memory mutation channels.

## Key Decisions (if any beyond L2)

**Hybrid rotation contract**: Rotation is explicit-first and threshold-fallback at turn boundaries, with typed reason codes and one-active-session enforcement.

**Replay-safe ingress semantics**: Duplicate message and rotate submissions collapse safely to one durable outcome using idempotency keys and transaction-safe reconciliation.

**Fail-closed surfaced contracts**: Added memory/rotation/timeline surfaces remain strict and contract-enforced, not permissive passthrough payloads.
