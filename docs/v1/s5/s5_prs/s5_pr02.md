# PR-02: Session Management Hardening + Memory Lifecycle Completion

## Goal

Complete Slice 5 trust/safety semantics by hardening message/session ingress and finishing deterministic rotation and memory lifecycle behavior.

## Builds On

Slice 5 PR-01 merged state (canonical memory item/revision model, explicit rotation, recall/projection contracts).

## Acceptance

- inferred memory remains `candidate` until explicit user promotion.
- correction/removal semantics are immediate and append-only (`validated` -> `superseded` / `retracted`).
- `POST /v1/sessions/{session_id}/message` supports idempotent replay using `Idempotency-Key`.
- conflicting idempotency reuse on the same session returns `409` with `E_IDEMPOTENCY_KEY_REUSED`.
- deterministic auto-rotation executes at turn boundaries with typed reasons:
  - `threshold_turn_count`
  - `threshold_age`
  - `threshold_context_pressure`
- one-active-session invariant remains preserved across explicit and threshold rotation paths.
- timeline supports incremental sync via `GET /v1/sessions/{session_id}/events?after={event_id}`.
- unknown timeline cursor returns `404` with `E_EVENT_CURSOR_NOT_FOUND`.
- context builder remains deterministic and bounded with fixed section order:
  1. `policy_system_instructions`
  2. `recent_active_session_turns`
  3. `rolling_session_summary`
  4. `durable_memory_recall`
  5. `open_commitments_and_jobs`
  6. `relevant_artifacts_and_signals`
- session surfaces include explicit lifecycle state (`active`, `rotating`, `closed`, `recovery_needed`).

## Non-Goals

- no generic memory CRUD write endpoint.
- no autonomous background memory rewrite/consolidation loop.
- no provider-coupled context compaction authority over canonical memory/session semantics.

## Notes

- context compaction is available only as an optimization seam via `ContextCompactionAdapter`; default behavior is noop.
- compaction must not alter canonical memory state, session authority, or surfaced action/memory audit invariants.
