# PR-01: URL Extraction Vertical + Safety/Reliability Hardening

## Goal

Deliver Slice 7 end-to-end with production-safe URL extraction, grounded responses, strict safety preflight, and release-gated reliability behavior.

## Builds On

Slice 6 PR-02 merged state (retrieval synthesis, provenance artifacts, typed capability runtime guardrails, and fail-closed egress preflight).

## Acceptance

- Ariel introduces `cap.web.extract` as an allowlisted `read` capability with no approval path.
- URL browsing executes through Ariel capability contracts (not model-vendor native browsing shortcuts), preserving provider portability boundaries.
- URL safety preflight is strict and fail-closed before extraction execution, including rejection of invalid/non-http(s) input and unsafe destination posture.
- Extraction egress remains constrained to explicit allowlisted destinations, and undeclared/malformed/non-allowlisted outbound intent is blocked before outbound calls.
- Successful URL requests return bounded structured extracted content plus grounded assistant response with inline citation markers and synchronized `assistant.sources[]`.
- Cited URL evidence is persisted as inspectable provenance artifacts with stable source identity and retrieval timing.
- Canonical source identity remains stable across redirects/retries so citation/provenance linkage is dedupe-safe and auditable.
- Invalid/restricted/unsupported/provider/runtime extraction failures surface stable typed outcomes with actionable recovery guidance and auditable lifecycle reasons.
- Large/complex pages remain bounded and explicit: truncation/partial coverage is surfaced clearly instead of silent timeout/degradation.
- Transient provider failures use bounded retry/backoff behavior without violating turn/runtime budgets or producing duplicate user-visible outcomes.
- Mixed turns containing `cap.web.extract` plus non-retrieval proposals keep retrieval-grounded assistant messaging while preserving structured lifecycle inspectability for all proposals.
- Regression coverage blocks release on browsing-specific safety and quality invariants: URL safety policy enforcement, bounded extraction behavior, citation/provenance correctness, and mixed-turn grounding integrity.

## Non-Goals

- does not add authenticated/sessioned browsing flows (login, cookie replay, paywall bypass, CAPTCHA solving).
- does not add browser automation or any side-effecting web action execution.
- does not add open-ended multi-hop crawling autonomy.
- does not add proactive/background URL monitoring behavior.

## Key Decisions (if any beyond L2)

**Single-PR delivery includes hardening**: This PR ships both vertical URL extraction behavior and production safety/reliability hardening, rather than staging hardening into a later PR.

**Read-only browsing boundary remains hard**: Slice 7 browsing is strictly `read` and evidence-oriented; no side-effecting web capability is introduced.
