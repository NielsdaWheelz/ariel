# PR-07: Deterministic Approval Expiry Reconciliation

## Goal

Ensure approval expiry reaches deterministic terminal state and user-visible lifecycle clarity even when no approval decision call is made.

## Builds On

PR-06 merged.

## Acceptance

- pending approvals that pass `expires_at` reconcile to terminal `expired` without requiring a `POST /v1/approvals` request.
- reconciled approvals no longer appear as indefinitely `pending` in user-facing timeline/action lifecycle views.
- reconciliation records exactly one auditable `evt.action.approval.expired` outcome linked to the original action attempt and approval reference.
- reconciled lifecycle state includes a clear expired reason consistent with Slice 2 expired-approval behavior.
- approval decision attempts against already-reconciled expired approvals stay non-executing and idempotent under existing resolved-approval conflict semantics, without duplicate side effects or duplicate expiry outcomes.
- surfaced lifecycle/action details remain redacted and contract-safe while showing the reconciled expired terminal state.

## Non-Goals

- does not add bulk, batched, or delegated approval workflows.
- does not change policy classification, taint semantics, or approval token rules.
- does not redesign scheduler architecture or introduce autonomous background side-effect loops.
- does not alter side-effect execution ordering/idempotency guarantees beyond expiry reconciliation semantics.

## Key Decisions (if any beyond L2)

**Expiry reconciliation is terminal-state convergence**: any path that encounters an expired pending approval must converge on the same terminal `expired` lifecycle outcome and audit chain.

**Exactly-once expiry observability is mandatory**: repeated reads or repeated approval decisions after reconciliation must not create additional expiry events or execution attempts.

**Resolved-approval conflict behavior remains stable**: once reconciliation marks an approval `expired`, later decision attempts stay terminal/non-executing under existing resolved-approval conflict semantics.
