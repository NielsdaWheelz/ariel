# PR-05: Surface-Only Lifecycle Contract + Approval Handle

## Goal

Close the remaining Slice 2 surface-boundary gap by making lifecycle payloads surfaced-only and approval-flow-safe without relying on internal records.

## Builds On

PR-04 merged.

## Acceptance

- user-facing turn/timeline payloads expose only the redacted, allowlisted lifecycle contract for action details; raw engine lifecycle internals are not included.
- approval-required lifecycle items include a stable surfaced approval reference that clients can use to submit approve/deny decisions.
- approval flows complete end-to-end using structured surfaced data only; clients do not parse assistant free text or read internal lifecycle records.
- phone-first timeline/action rendering and approval interactions consume surfaced lifecycle contracts only across inline reads and approval-gated paths.
- surfaced lifecycle details remain complete and redacted across proposal summaries, policy outcomes/reasons, approval outcomes/reasons, and execution outputs/errors.
- surfaced lifecycle coverage remains coherent for pending, denied, expired, approved, execution succeeded, and execution failed outcomes.
- regression coverage fails if raw action-attempt internals reappear in user-facing turn/timeline payloads or if approvals depend on non-surfaced fields.

## Non-Goals

- does not change Slice 2 policy decisions, taint semantics, approval token rules, or execution ordering/idempotency behavior.
- does not add new capability domains, background autonomy, or multi-user tenancy behavior.
- does not redesign the phone surface; only minimal UX changes needed for surfaced-contract approval continuity are in scope.
- does not lock all Slice 2 response schemas against leakage beyond this lifecycle/approval boundary (PR-06).

## Key Decisions (if any beyond L2)

**Stable approval handle is part of the surfaced contract**: approval actions are keyed by an opaque client-facing reference exposed in lifecycle data, so approval UX is decoupled from internal action-attempt record shapes.

**Surface lifecycle contract is authoritative for user-visible action state**: turn/timeline APIs treat surfaced lifecycle projection as the only supported lifecycle interface for clients.
