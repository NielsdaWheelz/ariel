# PR-02: Taint-Aware Policy + Capability Integrity + Egress Guardrails

## Goal

Harden Slice 2 action safety so untrusted content cannot silently escalate side effects, and execution remains integrity-checked and least-privilege.

## Builds On

PR-01 merged.

## Acceptance

- policy evaluation is taint-aware: side-effecting proposals materially influenced by untrusted external/tool content are escalated to approval or denied.
- capability execution is bound to proposal-time capability identity/version/contract metadata; mismatch at execution time is blocked and auditable.
- outbound network access for capability execution is constrained to policy-allowed destinations with deny-by-default semantics.
- side-effecting actions execute serially to preserve deterministic behavior and audit clarity.
- layered guardrails run before and after execution to prevent unsafe inputs/outputs from causing side effects or unsafe user-surfaced content.

## Non-Goals

- does not introduce dynamic plugin loading or arbitrary code-loading capability.
- does not expose generic shell/ssh execution to the model.
- does not add autonomous background action loops without user initiation.
- does not expand into multi-user tenancy or public internet hosting.

## Key Decisions (if any beyond L2)

**Security posture is architecture-first, not prompt-first**: prompt quality can help, but runtime policy, integrity checks, and egress controls are the actual safety boundary.

**Untrusted content is data, not authority**: tool/external outputs can inform planning but cannot independently grant permission to execute side effects.
