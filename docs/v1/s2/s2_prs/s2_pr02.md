# PR-02: Taint-Aware Authorization + Execution Integrity + Egress Controls

## Goal

Harden Slice 2 so untrusted content cannot escalate side effects and authorized actions execute only within integrity-checked, least-privilege runtime boundaries.

## Builds On

PR-01 merged.

## Acceptance

- when a side-effecting proposal is materially influenced by untrusted external or tool-returned content, policy does not auto-execute it inline; the outcome is explicit approval escalation or deny with auditable reason.
- proposal-time execution identity is captured and later enforced: if capability identity/version/contract metadata differs at execution time, execution is blocked before capability invocation and the integrity failure is auditable.
- outbound access during capability execution is deny-by-default and constrained to policy-allowlisted destinations; non-allowlisted egress attempts are blocked with explicit user-visible and auditable reasons.
- side-effecting action execution remains deterministic and serialized across approval-triggered runs, with idempotent execution boundaries that prevent duplicate side effects from replay/concurrency.
- pre-execution and post-execution guardrails can block unsafe inputs/outputs before side effects occur or before unsafe content is surfaced to the user.
- timeline/audit views remain reconstructable for taint-driven escalation/deny, integrity mismatch, egress deny, approval/expiry outcomes, execution success/failure, and guardrail-blocked outcomes.

## Non-Goals

- does not add agency- or calendar-domain workflows beyond framework-level safety probes.
- does not introduce dynamic plugin loading or generic shell/ssh model capabilities.
- does not add autonomous background side-effect loops.
- does not change single-user/private-hosting posture, tenancy model, or public ingress stance.

## Key Decisions (if any beyond L2)

**Hardening is enforced at runtime boundaries, not prompt behavior**: policy, integrity, egress, and guardrail checks are mandatory execution gates even when model output appears well-intentioned.
